## 五、核心业务流程测试

### 5.1 完整购物流程（P0）

```
小程序端：
  1. 浏览首页，查看商品
  2. 进入商品详情，查看图片和 SKU
  3. 选择颜色/尺码，加入购物车
  4. 进入购物车，勾选商品，去结算
  5. 确认收货地址（或新增地址）
  6. 选择优惠券（如有）
  7. 提交订单（模拟支付）

验证点：
  - 购物车数量正确
  - 订单金额计算正确（商品合计 + 运费 - 优惠券）
  - 订单状态为"待付款"(101)
  - 库存正确扣减
```

### 5.2 管理员发货流程（P0）

```
管理后台：
  1. 登录管理后台
  2. 进入订单管理 → 订单列表
  3. 筛选"待发货"订单
  4. 点击"发货"，选择物流公司，输入单号
  5. 确认发货

验证点：
  - 订单状态变为"已发货"(301)
  - 小程序端订单列表显示物流信息
  - 用户收到发货通知（如已配置）
```

### 5.3 用户确认收货（P0）

```
小程序端：
  1. 进入订单列表 → 待收货
  2. 点击订单，查看物流信息
  3. 点击"确认收货"

验证点：
  - 订单状态变为"已完成"(402)
  - 7 天后自动确认（由 task-order 定时任务处理）
```

### 5.4 超时未付款自动取消（P0）

```
自动化验证：
  1. 创建订单但不支付
  2. 等待 30 分钟（或临时修改超时时间为 1 分钟测试）
  3. task-order cancelUnpaid 定时触发

验证点：
  - 订单状态变为"已取消"(103)
  - 库存恢复
  - 优惠券释放（如有）
```

### 5.5 售后流程（P1）

```
小程序端：
  1. 进入已完成订单详情
  2. 点击"申请售后"
  3. 填写原因，提交申请

管理后台：
  4. 进入售后管理
  5. 审核通过/拒绝

验证点：
  - 售后状态正确更新
  - 拒绝时恢复流程清晰
```

### 5.6 优惠券流程（P1）

```
管理后台：
  1. 创建优惠券（满减/折扣）

小程序端：
  2. 下单时选择优惠券
  3. 验证优惠金额正确

验证点：
  - 优惠券在有效期内可使用
  - 金额计算正确
  - task-coupon 定时过期优惠券
```

### 5.7 场景/节日推荐流程（P2）

```
管理后台：
  1. 创建/编辑场景标签
  2. 绑定商品到场景
  3. 创建节日活动

小程序端：
  4. 首页查看场景推荐
  5. 活动位显示节日商品

验证点：
  - 场景推荐商品正确
  - 节日活动位优先级正确
```
