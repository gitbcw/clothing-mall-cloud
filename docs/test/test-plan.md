## 四、核心业务流程测试

### 4.1 完整购物流程

```
步骤 1: 登录小程序
步骤 2: 首页浏览商品
步骤 3: 进入商品详情
步骤 4: 选择颜色/尺码
步骤 5: 加入购物车
步骤 6: 进入购物车
步骤 7: 选择商品，去结算
步骤 8: 选择收货地址
步骤 9: 选择优惠券（如有）
步骤 10: 提交订单
步骤 11: 完成支付（模拟）
步骤 12: 查看订单状态

预期结果: 订单状态为"待发货"
```

### 4.2 管理员发货流程

```
步骤 1: 登录管理后台
步骤 2: 进入订单管理
步骤 3: 筛选"待发货"订单
步骤 4: 点击"发货"
步骤 5: 选择物流公司
步骤 6: 输入物流单号
步骤 7: 确认发货

预期结果: 订单状态变为"已发货"，小程序端显示物流信息
```

### 4.3 用户确认收货流程

```
步骤 1: 小程序进入订单详情
步骤 2: 查看物流信息
步骤 3: 点击"确认收货"
步骤 4: 确认弹窗点击"确定"

预期结果: 订单状态变为"已完成"
```

### 4.4 售后换货流程

```
步骤 1: 进入已完成订单详情
步骤 2: 点击"申请售后"
步骤 3: 选择换货商品
步骤 4: 选择新颜色/尺码
步骤 5: 填写换货原因
步骤 6: 提交申请

步骤 7: 管理后台进入售后管理
步骤 8: 查看售后申请
步骤 9: 审核通过

预期结果: 售后状态变为"已通过"
```

### 4.5 限时特卖流程

```
步骤 1: 管理后台创建限时特卖活动
步骤 2: 选择参与商品
步骤 3: 设置特卖价格和时间
步骤 4: 启用活动

步骤 5: 小程序首页查看限时特卖入口
步骤 6: 进入特卖商品详情
步骤 7: 确认显示特卖价格
步骤 8: 下单购买

预期结果: 订单按特卖价格计算
```
