# 小程序管理端功能迁移设计

> 日期：2026-04-06
> 状态：待审核
> 目标：让店主通过小程序完成日常运营，无需依赖电脑端后台管理

---

## 1. 背景与目标

当前系统有后台管理（Vue + Element UI，39 个控制器）和小程序管理端（3 个控制器，7 个页面）。小程序已覆盖商品管理、订单管理、场景管理，但大量日常运营功能仍需电脑操作。

**核心目标：** 店主绝大多数时候不用开电脑，在小程序端即可完成对小程序绝大部分功能的掌控。

---

## 3. 权限设计

### 3.1 角色定义

| 角色 | 职责 | 管理端入口 |
|------|------|-----------|
| **店主 (owner)** | 全部管理权限 | 完整 3-tab 管理界面 |
| **导购 (guide)** | 仅到店核销 | 单页核销界面（无 tab bar） |

### 3.2 权限矩阵

```
功能                    店主(owner)    导购(guide)
─────────────────────────────────────────────────
订单管理
  全部订单（发货/取消/退款/售后）    ✅            ❌
  到店核销                          ✅            ✅

商品管理（全部）                    ✅            ❌

设置页
  企微消息推送                      ✅            ❌
  搜索关键词管理                    ✅            ❌
  常见问题管理                      ✅            ❌
  商品分类管理                      ✅            ❌
  优惠券管理                        ✅            ❌
  场景管理                          ✅            ✅
  数据看板                          ✅            ❌
```

### 3.3 导购使用路径

进入管理端 → 核销单页 → 扫码/输入提货码 → 确认核销

- 不展示 tab bar（商品、设置 tab 隐藏）
- 后端校验：现有 `checkManager()` 已有 role 区分，扩展判断即可

## 5. 后端架构

### 5.1 设计原则

- **复用现有 Service 层**，不重复实现业务逻辑
- **新增 Manager 控制器为薄层**，调用现有 Service，添加权限校验
- 所有 manager 接口路径统一为 `/wx/manager/{resource}/*`

### 5.2 新增控制器

#### WxManagerWeWorkController (`/wx/manager/wework`)

```
GET    /tags           → 获取企微标签（复用 WeWork 逻辑）
GET    /pages          → 获取可选跳转页面
POST   /uploadMedia    → 上传图片到企微
POST   /sendCard       → 发送小程序卡片
POST   /sendMessage    → 发送纯文字消息
GET    /pushGroups     → 获取推送分组列表
```

#### WxManagerKeywordController (`/wx/manager/keyword`)

```
GET    /list           → 列表（复用 LitemallKeywordService）
POST   /create         → 创建
POST   /update         → 更新
POST   /delete         → 删除
```

#### WxManagerIssueController (`/wx/manager/issue`)

```
GET    /list           → 列表（复用 LitemallIssueService）
POST   /create         → 创建
POST   /update         → 更新
POST   /delete         → 删除
```

### 5.3 权限校验

所有新增 controller 统一使用：

```java
// 校验登录
checkManager();
// 校验店主角色（导购相关接口跳过此步）
if (!"owner".equals(user.getRole())) {
    return ResponseUtil.fail(403, "无权限");
}
```

## 6. 前端新增页面

### 6.1 企微消息推送

**pushSend** — 推送页
- 推送类型选择：小程序卡片 / 纯文字
- 卡片模式：上传封面图、标题、选择跳转页面
- 文字模式：文字内容输入
- 目标分组选择（checkbox 列表）
- 定时发送开关 + 时间选择
- 发送按钮

### 6.2 搜索关键词管理

**keywordManage** — 列表页
- 列表展示：关键词、排序号、启用状态
- 操作：新增行、编辑、删除、状态开关
- 一次性可编辑列表（类似 Excel 行内编辑），无需跳转编辑页

### 6.3 常见问题管理

**issueManage** — 列表页
- 列表展示：问题、排序号
- 操作：新增、编辑（跳转编辑页）、删除

**issueEdit** — 编辑页
- 问题输入
- 回答内容（富文本，支持图文）
- 排序数值
- 保存按钮

### 6.4 导购核销页

**verify** — 核销单页
- 提货码输入框（支持扫码输入）
- 显示订单信息（商品、金额）
- 确认核销按钮
- 核销成功/失败反馈
