## 二、一键部署（推荐）

### 2.1 本地执行

```bash
# 在本地项目根目录执行
./scripts/docker-start.sh deploy-prod
```

这会自动完成：
1. **本地打包**：后端 JAR + 管理前端
2. **上传文件**：JAR 和前端文件到服务器
3. **远程重启**：重建 Docker 镜像并启动服务

### 2.2 部署流程图

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  1.本地打包  │───►│ 2.上传 JAR  │───►│ 3.上传前端  │───►│ 4.远程重启  │
│  JAR + 前端  │    │   到服务器   │    │   到服务器   │    │   服务      │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

## 三、手动部署

### 3.1 完整部署命令

```bash
# 1. 本地打包后端
./scripts/docker-start.sh pack

# 2. 本地打包前端
./scripts/docker-start.sh pack-admin

# 3. 上传后端 JAR
scp docker/litemall/litemall.jar admin@47.107.151.70:/home/admin/clothing-mall/docker/litemall/

# 4. 上传前端
ssh admin@47.107.151.70 "rm -rf /home/admin/clothing-mall/docker/admin-dist"
scp -r docker/admin-dist admin@47.107.151.70:/home/admin/clothing-mall/docker/

# 5. 重建并重启
ssh admin@47.107.151.70 "cd /home/admin/clothing-mall/docker && docker compose build app && docker compose up -d app"
```

### 3.2 仅更新后端

```bash
# 本地打包
./scripts/docker-start.sh pack

# 上传并重启
scp docker/litemall/litemall.jar admin@47.107.151.70:/home/admin/clothing-mall/docker/litemall/
ssh admin@47.107.151.70 "cd /home/admin/clothing-mall/docker && docker compose build app --no-cache && docker compose up -d app"
```

### 3.3 仅更新前端

```bash
# 本地打包
./scripts/docker-start.sh pack-admin

# 上传（Nginx 自动生效，无需重启）
ssh admin@47.107.151.70 "rm -rf /home/admin/clothing-mall/docker/admin-dist"
scp -r docker/admin-dist admin@47.107.151.70:/home/admin/clothing-mall/docker/
```

## 六、常见问题

### 6.1 部署后代码不生效

**原因**：Docker 镜像未重建

**解决**：
```bash
ssh admin@47.107.151.70 "cd /home/admin/clothing-mall/docker && docker compose build app --no-cache && docker compose up -d app"
```

### 6.2 前端更新不生效

**原因**：浏览器缓存

**解决**：
1. 强制刷新：`Ctrl + Shift + R`
2. 或清除浏览器缓存
3. 或使用无痕模式

### 6.3 数据库连接失败

**排查步骤**：
```bash
# 1. 检查 MySQL 容器状态
ssh admin@47.107.151.70 "docker ps | grep mysql"

# 2. 检查 MySQL 日志
ssh admin@47.107.151.70 "docker logs clothing-mall-mysql --tail 50"

# 3. 测试连接
ssh admin@47.107.151.70 "docker exec clothing-mall-mysql mysqladmin ping -h localhost"
```

### 6.4 内存不足

**症状**：服务频繁重启

**解决**：调整 JVM 参数
```bash
# 编辑 docker-compose.yml 中的 JAVA_OPTS
JAVA_OPTS=-Xmx256m -Xms128m
```
