## 三、详细需求

### 3.1 促销管理合并（P0）

**现状**：限时特价、优惠券、满减为三个独立页面

**目标**：合并为一个页面，使用 Tab 切换

**需求点**：
- [x] 创建新的促销管理页面 `promotion/index.vue`
- [x] 三个 Tab：限时特价、优惠券、满减
- [x] 满减 Tab 默认隐藏（配置控制）
- [x] 保留原有页面功能，只做 UI 整合
- [x] 更新路由和菜单配置

**技术方案**：
- 使用 Element UI Tabs 组件
- 原有页面组件作为 Tab 内容嵌入
- 或使用 iframe 嵌入（简单但体验稍差）

---

### 3.2 自动预售逻辑（P0）

**业务规则**：
- 商品库存为 0 时，自动标记为「预售」状态
- 预售商品在小程序端显示「预售」标签
- 预售发货周期可配置（默认 2 天）

**需求点**：
- [x] 商品表增加 `is_presale` 字段
- [x] 系统配置增加 `presale_ship_days` 预售发货天数
- [x] 下单时扣减库存，库存为 0 时自动设置 `is_presale = true`
- [x] 补货时（库存 > 0）自动取消预售状态
- [x] 小程序商品详情显示预售标签和预计发货时间

**技术方案**：
- 数据库迁移脚本
- `LitemallGoodsService` 增加预售逻辑
- 小程序商品详情页增加预售展示

---

### 3.3 首页数据看板（P1）

**现状**：Dashboard 只有基础统计

**目标**：增加用户数、活跃数、转化率模块

**需求点**：

#### 用户数模块
- [x] 总用户数
- [x] 今日新增用户
- [x] 7 日新增趋势图

#### 活跃数模块
- [x] 日活（DAU）- 今日活跃用户数
- [x] 周活（WAU）- 近 7 日活跃用户数
- [x] 月活（MAU）- 近 30 日活跃用户数

#### 转化率模块
- [x] 推送 → 查看率（推送消息被打开的比例）
- [x] 商品详情 → 下单率
- [x] 收藏量统计

**技术方案**：
- 利用现有埋点数据（`litemall_tracker` 表）
- 后端增加统计 API
- 前端使用 ECharts 展示趋势图

---

### 3.4 推送组管理（P1）

**现状**：企微推送只能推送给所有用户

**目标**：支持用户分组推送

**需求点**：
- [x] 创建推送组表 `litemall_push_group`
- [x] 系统配置增加 `active_days`（活跃判定天数，默认 3）和 `dormant_days`（潜水判定天数，默认 30）
- [x] 四种分组类型：
  - **测试组**：手动添加成员（蜜姐、开发组）
  - **活跃组**：近 N 天有浏览记录的用户（自动判定，N 可配置，默认 3 天）
  - **打捞组**：每次推送时从潜水组随机抽取 20 人
  - **潜水组**：超过 N 天未活跃的用户（自动判定，N 可配置，默认 30 天）
- [x] 推送时选择目标组
- [x] 推送组管理页面

**技术方案**：
- 数据库表设计
- 定时任务更新活跃/潜水组
- 推送接口增加分组参数

---

### 3.5 定时推送（P1）

**现状**：推送只能立即发送

**目标**：支持定时发送

**需求点**：
- [x] 推送表增加 `scheduled_at` 字段
- [x] 推送页面增加定时发送选项
- [x] 后端定时任务检查并发送
- [x] 推送记录页面显示发送状态

**技术方案**：
- 使用 Spring `@Scheduled` 定时任务
- 每分钟检查一次待发送的推送

### 3.8 广告位优先级（P2）

**业务规则**：
- 优先级：节假日款 > 特价商品 > 本周上新

**需求点**：
- [x] 分类表增加 `is_holiday` 字段（手动配置）
- [x] 广告位配置页面增加优先级设置
- [x] 小程序首页按优先级展示商品
