## 一、管理端用户行为

### 1.1 上新

* **上新周期**
  * 每年**冬春两季**集中上新（主要上新季）
  * 其余时间偶尔上线新款
* **上架方式**
  * 在现有管理端小程序基础上扩展上架功能
  * 方便在实体店内用手机操作上架

### 1.2 订单处理

* **订单信息 → 处理**
  * 实体打包
  * 发货
* **售后信息 → 处理**

### 1.3 海报/活动

* 每天/晚上推送
* 通过**企业微信**推送小程序卡片消息

### 1.4 清货

* **实体店清货**：换季时实体店特价清货
* **线上通知**：通过推送告知用户线下清货活动，吸引用户到店
* **线上清货专区**：暂不开发，待后续评估

### 1.5 售卖后系统操作

* **自动预售**：商品库存为 0 时，系统自动标记为「预售」状态
* **补货周期**：预售商品等待两天后补货发货

## 二、管理端功能规划总览

> **重要说明**：这是**全新的菜单结构规划**，现有后台菜单需要基于此进行重构。
> 详细评估见：[菜单迁移评估.md](菜单迁移评估.md)

### 目标菜单结构

| 模块 | 需求点数 | 优先级 | 技术要点 | 现有菜单合并来源 |
|------|----------|--------|----------|------------------|
| 首页 | 14 | P1 | 数据统计、趋势图 | Dashboard 增强 |
| 订单管理 | 8 | P0 | 订单状态流转 | 订单管理（保留） |
| 商品管理 | 37 | P0 | AI 标签识别、自动预售 | 商品管理（合并） |
| 平台设置 | 28 | P1 | 定时推送、用户分组 | 会员+活动+运营+系统+小程序配置 |
| 营收分析 | 11 | P2 | 报表统计 | 数据统计（保留） |

### 菜单重构原则

1. **刚需保留**：核心业务页面（订单、商品）保留原有功能
2. **合理合并**：配置类页面合并到「平台设置」
3. **删除冗余**：低使用率功能评估后删除或隐藏
4. **分阶段实施**：先菜单重组，后功能增强

## 五、商品管理

> **核心流程**：拍照录入 → SKU 入库 → 选择上架 → 补充信息 → 确认上架

### 5.1 SKU 管理（新增）

#### 拍照录入

* **移动端拍照**：在店铺内用手机拍照
* **AI 自动识别**：分类、场景、生成简介
* **产生 SKU**：拍照后自动创建 SKU 记录

#### SKU 状态

| 状态 | 说明 |
|------|------|
| 待处理 | 刚录入，照片可能需精修 |
| 已上架 | 已完成上架，小程序可见 |
| 已下架 | 已下架，库存保留 |

#### SKU 列表

* 支持按状态筛选
* 支持搜索（款号/编号）
* 支持批量操作

### 5.2 上架流程

#### 入口方式

1. **从 SKU 上架**：SKU 列表 → 选择 → 补充信息 → 上架
2. **搜索上架**：输入款号/编号 → 找到 SKU → 补充信息 → 上架
3. **浏览库存**：浏览 SKU 库 → 选择 → 补充信息 → 上架

#### 上架信息

| 字段 | 必填 | 说明 |
|------|------|------|
| 商品名称 | 是 | AI 生成，可修改 |
| 商品编号 | 否 | 与实体店编号对齐 |
| 商品分类 | 是 | AI 推荐，可修改 |
| 场景标签 | 否 | AI 推荐，可多选 |
| 商品简介 | 否 | AI 生成，可修改 |
| 正常价 | 是 | 商品售价 |
| 特价 | 否 | 设置后自动标记为特价商品 |
| 库存 | 是 | 商品库存数量 |
| 商品图片 | 是 | 可精修、调整顺序 |

#### 上架确认

* 预览商品信息
* 确认上架
* 小程序端立即可见

### 5.3 广告位说明

* **展示优先级**（自动为主，支持手动调整）：
  1. 节假日时优先展示节假日款（场景标签为节假日）
  2. 有特价商品时优先展示特价商品
  3. 正常情况展示本周上新（商品图片轮动）
* 支持管理员手动置顶/调整

### 5.4 快捷下架

* 支持按分类批量下架（换季场景）
* 支持单个商品下架
* 下架后 SKU 保留，状态变为「已下架」

### 5.5 分类

| 分类 | 说明 | 季节开关 |
|------|------|----------|
| 上装内搭 | 默认内置 | 可配置 |
| 上装外搭 | 默认内置 | 可配置 |
| 裤装系列 | 默认内置 | 可配置 |
| 裙装系列 | 默认内置 | 可配置 |
| 套装系列 | 默认内置 | 可配置 |

> **季节开关说明**：每个分类可配置在特定季节/时期是否在小程序显示。例如夏季可隐藏冬装分类。

### 5.3 场景

场景类似商品的「分类标签」，用于筛选商品。海报在后台广告管理中手动配置关联。

| 场景 | 海报配置 | 空场景处理 |
|------|----------|------------|
| 轻松通勤 | 手动关联海报 | - |
| 户外游玩 | 手动关联海报 | 场景下无商品时不轮播 |
| 文艺聚会 | 手动关联海报 | 内置主题，有开关控制 |

## 六、平台设置

### 6.1 推送设置

#### 推送方式

* **企业微信小程序卡片消息**

#### 推送内容

* **海报精照**：精选照片海报
* **简文/活动消息**：文字+图片的活动通知

#### 定时推送

* 支持**可配置的定时推送**功能
* 默认示例：中午 12:00
* 可配置其他时间点

### 6.2 推送组

#### 测试组

* 成员：蜜姐、开发组
* 用途：海报、活动推送先发此组确认无误后再发用户组

#### 活跃组

* **自动判定**：近三天有浏览记录的用户
* 支持手动添加
* 可查看浏览轨迹

#### 打捞组

* **随机抽取**：每次从潜水组中随机抽取 20 个用户
* 支持手动添加
* 用于激活沉默用户

#### 潜水组

* 沉默用户池
* 作为打捞组的用户来源

### 6.3 用户列表

#### 用户角色

| 角色 | 权限说明 |
|------|----------|
| 普通用户 | 正常浏览、下单 |
| 店主/运营 | 可切换到扩展功能（管理端小程序） |

#### 用户信息

* 基本信息
* 收货地址

### 6.4 广告管理

| 功能 | 说明 |
|------|------|
| 广告位 | 首页轮播图、场景海报配置 |
| 专题管理 | 内容专题页面 |
| 穿搭推荐 | 穿搭内容推荐 |

### 6.5 促销管理

> **设计说明**：限时特价、优惠券、满减合并到同一页面，使用 Tab 切换。满减默认隐藏，可通过配置启用。

| Tab | 状态 | 说明 |
|-----|------|------|
| 限时特价 | 显示 | 商品特价促销 |
| 优惠券 | 显示 | 优惠券发放管理 |
| 满减 | **隐藏** | 未来可能启用 |

### 6.6 活动管理

* 活动创建与配置
* 活动效果追踪

### 6.7 店铺设置

* 门店简介
* 门店位置
* 门店人员
