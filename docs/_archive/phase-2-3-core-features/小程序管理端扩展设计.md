# 小程序管理端扩展设计

> **文档目的**：定义小程序管理端扩展的功能范围、交互流程和技术方案
> **更新时间**：2026-03-21
> **状态**：草稿 - 待讨论完善

## 一、背景与目标

### 1.1 背景

- **实体店场景**：店主/导购在店铺内需要快速上架商品、处理订单
- **现有痛点**：Web 管理后台在手机上操作不便，上架商品需要多步操作
- **机会**：已有小程序用户端，可扩展管理功能给店主/运营使用

### 1.2 目标

| 目标 | 说明 |
|------|------|
| 快速上架 | 手机拍照 → AI 识别 → 一键上架 |
| 订单处理 | 随时随地查看和处理订单 |
| 轻量推送 | 简化的海报推送功能 |

### 1.3 用户角色

| 角色 | 使用场景 |
|------|----------|
| 店主 | 上架商品、查看数据、配置推送 |
| 导购 | 上架商品、处理订单 |

## 二、入口设计

### 2.1 方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| **A. 同一小程序扩展** | 无需下载、用户数据共享 | 功能入口需隐藏、可能影响用户体验 |
| **B. 独立管理小程序** | 功能独立、权限清晰 | 需要单独开发、用户需切换小程序 |
| **C. 企业微信应用** | 与企微推送打通、适合运营 | 开发成本高、店主可能不用企微 |

### 2.2 采用方案：同一小程序扩展

**入口逻辑**：
1. 普通用户：看到标准商城界面
2. 店主/运营：「我的」页面显示「管理入口」按钮，点击进入管理模式

**入口设计**：
```
「我的」页面
├─ 用户信息
├─ 我的订单
├─ ...
└─ 【管理入口】← 仅店主/运营可见
```

### 3.2 快速上架

> **核心流程**：拍照 → 产生 SKU → 选择上架 → 补充信息 → 确认上架

#### 3.2.1 流程说明

**两阶段流程**：

```
阶段一：拍照录入（产生 SKU）
┌─────────┐    ┌─────────┐    ┌─────────┐
│  拍照   │───►│ AI 识别 │───►│ SKU 入库 │
│         │    │         │    │         │
└─────────┘    └─────────┘    └─────────┘

阶段二：选择上架
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│浏览/搜索│───►│ 选择SKU │───►│补充信息 │───►│ 确认上架 │
│  SKU    │    │         │    │         │    │         │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
```

**为什么分两阶段**：
- 拍照时照片可能需精修，不能直接上架
- 需要补充完整信息才能上架
- SKU 可多次上架/下架

#### 3.2.2 阶段一：拍照录入

#### 3.2.2 阶段一：拍照录入

**步骤 1：拍照/选图**

| 功能 | 说明 |
|------|------|
| 拍照 | 调用相机拍摄商品照片 |
| 相册 | 从相册选择已有照片 |
| 多图 | 支持选择多张（主图+详情图） |

**步骤 2：AI 识别**

| 识别项 | 说明 | 用户操作 |
|--------|------|----------|
| 商品分类 | 识别上装/裤装/裙装等 | 可手动选择 |
| 场景标签 | 识别通勤/户外/聚会等 | 可多选 |
| 商品简介 | 生成商品描述文案 | 可编辑 |
| 商品参数 | 推荐尺码、材质等 | 可勾选 |

**技术方案**：
- 调用多模态大模型 API（如 GPT-4V、Claude、通义千问等）
- 通过精心设计的提示词（Prompt）实现识别
- 后续可优化提示词提升识别准确率

> **待选型**：具体使用哪个大模型 API，需评估成本和效果

**步骤 3：确认 SKU 信息**

| 字段 | 必填 | 说明 |
|------|------|------|
| 商品编号 | 否 | 与实体店编号对齐，方便后续检索 |
| 商品分类 | 是 | AI 推荐，可修改 |
| 场景标签 | 否 | AI 推荐，可多选 |

**步骤 4：SKU 入库**

- 预览识别结果
- 点击「保存」入库
- SKU 状态变为「待处理」
- 可继续拍照录入下一个

#### 3.2.3 阶段二：选择上架

**入口方式**：

| 入口 | 说明 |
|------|------|
| SKU 列表 | 浏览 SKU 库，选择要上架的 |
| 搜索 | 输入款号/编号，快速找到 SKU |
| 待处理提醒 | 从首页待处理入口进入 |

**上架信息补充**：

| 字段 | 必填 | 说明 |
|------|------|------|
| 商品名称 | 是 | AI 生成，可修改 |
| 商品简介 | 否 | AI 生成，可修改 |
| 正常价 | 是 | 商品售价 |
| 特价 | 否 | 设置后自动标记为特价商品 |
| 库存 | 是 | 商品库存数量 |
| 商品图片 | 是 | 可调整顺序、标记主图 |

**上架确认**：
- 预览商品完整信息
- 点击「上架」
- 小程序端立即可见

#### 3.2.4 暂存功能

**目的**：帮用户省时间，避免重复填写

| 场景 | 处理 |
|------|------|
| 拍照录入中途退出 | 自动保存当前内容 |
| 下次进入录入 | 提示恢复上次内容 |
| SKU 入库成功 | 清除草稿 |

**实现方式**：
- 使用小程序本地存储（localStorage）暂存
- 存储内容：已填写的字段 + 已选图片

### 3.5 快捷推送

#### 3.5.1 功能范围

| 功能 | 说明 |
|------|------|
| 选择海报 | 从已上传的海报中选择 |
| 选择推送组 | 测试组 → 确认无误 → 活跃组/打捞组 |
| 发送操作 | 调用企微推送接口 |

#### 3.5.2 推送流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 选择海报 │───►│ 选择推送组│───►│ 发送到  │───►│ 确认效果 │
│         │    │         │    │ 测试组  │    │         │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
                                    │
                                    │ 确认无误后
                                    ▼
                              ┌─────────┐
                              │ 发送到  │
                              │ 用户组  │
                              └─────────┘
```

> **重要**：推送必须先发测试组，确认效果后再发用户组（活跃组/打捞组）

#### 3.5.3 推送记录

- 推送数据记录到数据库
- 暂不做统计页面，后续需要时再开发

## 四、权限控制

### 4.1 用户角色

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| 普通用户 | 正常购物用户 | 无管理入口 |
| 店主/运营 | 管理人员 | 全部管理功能 |

### 4.2 角色判断方式

- 在 Web 管理后台「用户列表」中标记用户角色
- 小程序登录时通过 API 获取用户角色
- 「我的」页面根据角色显示/隐藏「管理入口」

### 4.3 功能权限矩阵

| 功能 | 店主 | 导购 |
|------|------|------|
| 快速上架 | ✅ | ✅ |
| 订单处理 | ✅ | ✅ |
| 数据概览 | ✅ | ✅（仅查看） |
| 快捷推送 | ✅ | ❌ |
| 价格修改 | ✅ | ❌（或需审批） |
