## 二、拉新体系（多渠道获客）

### 2.1 到店转化

```
顾客到店
    │
    ↓
店员/老板引导扫码
    │
    ↓
添加企业微信 ──→ 自动推送欢迎语 + 新人权益
    │
    ↓
点击注册链接 ──→ 小程序注册
    │
    ↓
系统记录归因（门店/店员）
```

**价值**：
- 客户资产沉淀
- 自动化欢迎流程
- 归因追踪（评估各渠道效果）

### 2.2 裂变推荐

```
老顾客生成专属海报/链接
    │
    ↓
分享到朋友圈/微信群
    │
    ↓
新顾客扫码注册
    │
    ↓
系统识别推荐关系
    │
    ↓
双方获得奖励
    ├── 老顾客：优惠券/积分/现金奖励
    └── 新顾客：新人专属优惠
```

**价值**：
- 低成本获客
- 社交信任背书
- 可追踪裂变效果

### 2.3 活动引流

```
老板配置活动（新品/促销/节日）
    │
    ↓
生成活动海报（带小程序码）
    │
    ↓
发布到朋友圈/社群/企微推送
    │
    ↓
新顾客扫码进入 ──→ 注册转化
```

## 三、活跃体系（促活与留存）

### 3.1 会员分层

```
顾客行为数据
    │
    ├── 消费金额 → 会员等级（普通/VIP/SVIP）
    ├── 消费频次 → 活跃度标签
    ├── 浏览偏好 → 兴趣标签
    └── 最后活跃时间 → 流失预警
```

**价值**：
- 精准运营：不同人群不同策略
- 预警流失：及时挽回

### 3.2 消息触达

| 场景 | 触达方式 | 内容示例 |
|------|----------|----------|
| 新品上架 | 企微推送 | "本周新品已上线，点击查看" |
| 活动提醒 | 企微推送 | "限时折扣，最后3天" |
| 生日关怀 | 企微推送 | "生日快乐！专属优惠券已发放" |
| 流失唤醒 | 企微推送 | "好久不见，想念你" |
| 订单通知 | 小程序订阅消息 | "您的订单已发货" |

### 3.3 积分体系

```
消费 → 获得积分
    │
    ↓
积分用途
    ├── 兑换优惠券
    ├── 兑换礼品
    └── 抵扣现金
```

## 四、交易体系（变现闭环）

### 4.1 完整购物流程

```
┌─────────────────────────────────────────────────────────────────┐
│  浏览 → 搜索 → 详情 → 选规格 → 加购 → 结算 → 支付 → 订单       │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 配送方式

| 方式 | 流程 | 适用场景 |
|------|------|----------|
| **快递发货** | 下单 → 备货 → 发货 → 物流追踪 → 签收 | 外地顾客 |
| **到店自提** | 下单 → 备货 → 通知 → 到店 → 核销取货 | 本地顾客、试穿需求 |
| **同城配送** | 下单 → 备货 → 第三方配送 → 送达 | 同城急件（可后置） |

### 4.3 库存管理

```
商品入库（进货）
    │
    ↓
设置SKU库存（颜色 × 尺码）
    │
    ↓
销售自动扣减库存
    │
    ↓
库存预警 → 补货提醒
```

## 五、履约体系（交付保障）

### 5.1 订单状态机

```
                              ┌─────────────────────────────────────┐
                              │           配送方式分流              │
                              └─────────────────────────────────────┘
                                          │
            ┌─────────────────────────────┼─────────────────────────────┐
            ↓                             ↓                             ↓
      ┌──────────┐                 ┌──────────┐                 ┌──────────┐
      │ 快递发货 │                 │ 到店自提 │                 │ 同城配送 │
      └──────────┘                 └──────────┘                 └──────────┘
            │                             │                             │
            ↓                             ↓                             ↓
        已发货                         待自提                         待配送
            │                             │                             │
            ↓                             ↓                             ↓
        待收货                         已自提                         配送中
            │                             │                             │
            ↓                             ↓                             ↓
        已完成                         已完成                         已完成


基础状态流转：
待支付 → 已支付 → [配送方式分流] → 已完成
                      │
                      ↓
                   已取消
                      │
                      ↓
                   售后中 → 已退款
```

### 5.2 发货流程（快递）

```
订单支付成功（配送方式：快递发货）
    │
    ↓
通知老板/运营（企微/站内消息）
    │
    ↓
备货（检查库存、打包）
    │
    ↓
发货方式
    ├── 自有库存：直接发货
    └── 供应商协作：通知供应商发货
    │
    ↓
录入物流单号
    │
    ↓
顾客收到发货通知 → 查看物流
```

### 5.3 自提流程（到店）

```
订单支付成功（配送方式：到店自提）
    │
    ↓
通知老板/运营（企微/站内消息）
    │
    ↓
备货（检查库存、打包、标记预留）
    │
    ↓
通知顾客（企微/订阅消息）
    │
    ├── "您的商品已备好，请于X日内到店自提"
    └── 附：门店地址、营业时间、自提码
    │
    ↓
顾客到店
    │
    ↓
店员核销（扫描自提码 / 输入手机号后四位）
    │
    ↓
系统自动确认收货 → 订单完成
    │
    ↓
可选：引导顾客试穿、连带销售
```

**超时处理**：
- 自提有效期（如7天）到期未取 → 自动提醒 → 仍未取 → 联系顾客处理

### 5.4 同城配送流程

```
订单支付成功（配送方式：同城配送）
    │
    ↓
通知老板/运营（企微/站内消息）
    │
    ↓
备货（检查库存、打包）
    │
    ↓
发起配送（对接第三方配送平台）
    │
    ├── 美团跑腿
    ├── 顺丰同城
    ├── 蜂鸟即配
    └── 达达快送
    │
    ↓
系统同步配送状态
    │
    ├── 骑手接单 → 顾客可见骑手信息
    ├── 骑手取货 → 顾客收到提醒
    └── 配送中 → 实时位置追踪（可选）
    │
    ↓
骑手送达 → 顾客确认签收
    │
    ↓
系统自动确认收货 → 订单完成
```

**配送费计算**：
- 按距离/重量自动计算
- 满额免运费配置
- 顾客下单时预付配送费

**异常处理**：
- 骑手取消 → 重新发单
- 顾客拒收 → 退回门店 → 联系处理

## 六、售后体系（服务保障）

### 6.1 售后政策

**川着 transmute 采用「换货为主」模式**：
- 主推换货流程，顾客优先选择换货
- 保留退款功能，用于特殊情况处理
- 由运营人工判断是否允许退款

### 6.2 售后类型

| 类型 | 场景 | 流程 | 是否支持 |
|------|------|------|----------|
| **换货** | 尺码不合适/颜色差异 | 申请 → 审核 → 寄回/到店 → 换货 | ✅ 主流程 |
| **退款** | 特殊情况 | 申请 → 审核 → 退款 | ✅ 特殊情况 |
| **维修** | 质量问题（可修） | 申请 → 寄回 → 维修 → 寄回 | ✅ 按需 |

### 6.3 换货流程（核心流程）

```
顾客发起换货申请（小程序）
    │
    ├── 选择换货商品（同款不同规格）
    │   └── 如需补差价：显示差价金额
    │
    ↓
老板/运营审核
    │
    ├─ 拒绝：填写原因 → 顾客可见
    │
    └─ 同意：
        │
        ↓
    换货方式
    ├── 到店换货：顾客携带商品到店 → 现场换货 → 系统核销
    └── 寄回换货：
        │
        ↓
    顾客寄回原商品（填写物流单号）
        │
        ↓
    老板验货确认
        │
        ├── 商品完好：发出新商品
        └── 商品有损：拒绝换货 / 协商处理
        │
        ↓
    换货完成 → 订单状态更新
```

### 6.4 换货差价处理

| 情况 | 处理方式 |
|------|----------|
| 换同价商品 | 无需补差 |
| 换高价商品 | 顾客补差价（微信支付） |
| 换低价商品 | 差价退回（可选，商家配置） |

### 6.5 售后流程（标准模式 - 含退款）

```
顾客发起申请（小程序/企微）
    │
    ↓
老板/运营审核
    │
    ├─ 拒绝：填写原因 → 顾客可见
    │
    └─ 同意：
        │
        ↓
    提供退货地址
        │
        ↓
    顾客寄回（填写单号）
        │
        ↓
    老板验货确认
        │
        ↓
    系统自动退款（微信原路）
```
