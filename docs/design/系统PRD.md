## 一、系统概述

### 1.1 业务背景

**川着 transmute** 是一家女装精品服装店，主打每季特色款式，目标客群为高知女性白领（20-45岁）。

店铺目前以线下实体店为主，希望拓展线上渠道，实现：
- 线上展示商品，吸引顾客
- 支持到店自提或快递发货
- 通过企业微信沉淀会员、触达用户

### 1.2 系统定位

川着 transmute 线上渠道系统是一套 **"线下实体店 + 微信生态 + 线上商城"** 的一体化业务系统。

**核心目标**：
- 通过 **企业微信** 完成用户沉淀与触达
- 通过 **小程序商城** 完成交易闭环
- 通过 **运营后台** 管控商品、活动、订单、售后

### 1.3 核心能力地图

```
用户沉淀 → 商品展示 → 下单支付 → 履约交付 → 售后服务
    │          │          │          │          │
    ▼          ▼          ▼          ▼          ▼
 企业微信    小程序商城   微信支付   快递/自提   换货为主
```

### 1.4 系统边界

| 模块 | 是否系统内 | 说明 |
|------|-----------|------|
| 商品管理 | ✅ | 运营后台配置 |
| 交易支付 | ✅ | 微信支付 |
| 会员管理 | ✅ | 简化版，关注即会员 |
| 营销活动 | ✅ | 优惠券、满减、限时特卖 |
| 售后服务 | ✅ | 主推换货，保留退款 |
| 物流调度 | ❌ | 人工对接 |
| 供应商协作 | ❌ | 人工通知发货 |
| 会员等级/积分 | ❌ | 暂不需要 |

## 四、核心业务流程

### 4.1 用户沉淀流程

```
顾客到店消费
    │
    ▼
店员引导扫码关注企业微信
    │
    ▼
系统自动记录为会员（无等级体系）
    │
    ▼
企微推送：新人优惠 + 商城入口
```

**说明**：
- 关注企业微信 = 成为会员
- 不需要会员等级、积分体系
- 后台可累计消费数据（备用）

### 4.2 商品上架流程

```
店主选品/进货
    │
    ▼
运营在后台录入商品
    ├── 商品信息（名称、描述、分类）
    ├── 商品图片（支持多图）
    ├── 价格（售价、原价）
    └── SKU规格（颜色 × 尺码）
    │
    ▼
商品上架 → 小程序自动展示
```

**库存特点**：
- 衣服一般只有1-2件
- 裤子可能多尺码
- 不累库，库存量小

### 4.3 下单支付流程

```
用户浏览商品（小程序）
    │
    ▼
选择规格（颜色/尺码）→ 加入购物车
    │
    ▼
确认订单
    ├── 选择配送方式（快递/到店自提）
    ├── 选择/填写收货地址
    ├── 使用优惠券（如有）
    └── 查看实付金额
    │
    ▼
微信支付 → 订单生成
```

### 4.4 订单履约流程

```
订单支付成功
    │
    ├── 配送方式：快递发货
    │       │
    │       ▼
    │   运营通知供应商发货（人工）
    │       │
    │       ▼
    │   供应商发货 → 运营录入快递单号
    │       │
    │       ▼
    │   用户查看物流 → 确认收货
    │
    └── 配送方式：到店自提
            │
            ▼
        运营备货 → 通知用户到店
            │
            ▼
        用户到店 → 店员核销 → 订单完成
```

### 4.5 售后服务流程

**默认政策：只换不退**

```
用户申请售后（小程序）
    │
    ▼
选择换货规格（颜色/尺码）
    │
    ▼
系统自动计算差价
    ├── 无差价 → 直接换货
    ├── 需补差价 → 用户支付差价
    └── 可退差价 → 系统退款
    │
    ▼
运营审核
    │
    ├── 审核通过
    │       │
    │       ├── 到店换货：用户携带商品到店 → 现场换货
    │       └── 寄回换货：用户寄回 → 运营验货 → 发出新货
    │
    └── 审核拒绝：填写原因 → 用户可见
```

**特殊情况退款**：
- 保留退款功能，用于特殊情况处理
- 由运营人工判断是否允许退款

### 4.6 营销活动流程

```
运营配置活动（后台）
    │
    ├── 新人优惠：首单专享优惠
    ├── 生日七折：检测会员生日 → 自动发放优惠券
    ├── 限时特卖：商品限时特价
    └── 满减活动：订单满减规则
    │
    ▼
企业微信推送活动消息
    │
    ▼
用户点击 → 进入商城 → 下单
```

### 4.7 实体店销售同步

```
线下店销售商品
    │
    ▼
导入销售记录到系统
    │
    ▼
系统自动下架对应商品
    │
    ▼
线上线下库存同步
```
