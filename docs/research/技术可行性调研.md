## 一、拉新体系

### 1.1 到店转化流程

**业务流程**：
```
顾客到店 → 店员引导扫码（企微二维码）→ 添加企微 → 自动推送欢迎语+新人优惠 → 点击注册 → 小程序注册 → 归因记录
```

**技术验证结果**：

| 环节 | 可行性 | 说明 |
|------|--------|------|
| 企微二维码 | ✅ | 支持生成带参数二维码，可区分门店/导购 |
| 添加企微 | ✅ | 无限制 |
| 自动欢迎语 | ✅ | 支持文字+图片+小程序链接，最多9个附件 |
| 欢迎语带小程序链接 | ✅ | 支持，需填写 appid、路径等参数 |
| 用户归因（企微↔小程序） | ⚠️ | **需绑定微信开放平台**，通过 UnionID 关联 |
| 新人优惠券发放 | ✅ | 小程序内实现，微信官方不收费 |

**关键限制**：
1. **UnionID 关联**：企业微信和小程序必须绑定到同一个微信开放平台账号，才能获取 UnionID 实现用户关联
2. **欢迎语文字限制**：建议控制在200字符以内
3. **企业认证**：未认证企业成员上限200人，认证后无上限（300元/年）

**费用**：
- 企业微信认证：**300元/年**
- 欢迎语推送：免费
- 优惠券发放：免费（需小程序支持该功能）

**参考资料**：
- [企业微信官方 - 发送新客户欢迎语](https://developer.work.weixin.qq.com/document/90001/90143/92599)
- [企业微信 - 获取客户详情](https://work.weixin.qq.com/api/doc/90001/90143/92265)

### 1.2 裂变推荐流程

**业务流程**：
```
老顾客生成专属海报/链接 → 分享 → 新顾客扫码注册 → 系统识别推荐关系 → 双方获得奖励
```

**技术验证结果**：

| 环节 | 可行性 | 说明 |
|------|--------|------|
| 专属小程序码生成 | ✅ | 使用 `wxacode.getUnlimited` 接口，**数量无限制** |
| 参数传递（推荐人ID） | ✅ | 通过 scene 参数传递，最长32个可见字符 |
| 推荐关系识别 | ✅ | 新用户扫码时解析 scene 获取推荐人ID |
| 奖励发放 | ✅ | 优惠券/积分在小程序内实现 |

**关键限制**：
1. `wxacode.getUnlimited` 的 scene 参数最长32个可见字符
2. 小程序码永久有效，无法设置过期时间
3. 小程序码宽度最小280px，建议430px

**费用**：
- 小程序码生成：免费（无数量限制）

**参考资料**：
- [微信官方 - 获取不限制的小程序码](https://developers.weixin.qq.com/miniprogram/dev/server-API/qrcode-link/qr-code/api_getunlimitedqrcode)

## 二、消息触达

### 2.1 企业微信推送

**技术验证结果**：

| 项目 | 说明 |
|------|------|
| 推送消息类型 | 文字、图片、小程序、视频、文件等 |
| 频率限制 | 动态调整，每位客户每月最多接收「当月天数」条 |
| 单次发送量 | 最多2万个客户、200个客户群 |
| 审核要求 | 无需审核 |

**关键限制**：
1. 频率由企业微信动态调整，与「企业活跃度、用户互动」相关
2. 客户可在设置中调整接收频率（每周7条或每天1条）
3. 超过接收上限的客户无法收到群发

**费用**：免费

**参考资料**：
- [企业微信官方 - 创建企业群发](https://work.weixin.qq.com/api/doc/90001/90143/92698)

### 2.2 小程序订阅消息

**技术验证结果**：

| 项目 | 说明 |
|------|------|
| 订阅类型 | 一次性订阅（7天有效）、长期订阅（需特殊资质） |
| 授权方式 | 用户主动点击按钮触发 |
| 模板来源 | 公共模板库 或 自定义模板（需审核） |
| 2026新规则 | 支付后可免弹窗订阅（使用支付订单号作为code） |

1. 一次性订阅：用户授权一次，只能发一条消息，7天内有效

3. 内容禁止纯营销，需与服务场景强关联
4. 模板需审核通过后使用

**适用场景**：
- 订单支付成功通知
- 发货通知
- 售后进度通知

**费用**：免费

## 七、风险点与建议

### 7.1 技术风险

| 风险点 | 影响 | 应对方案 |
|--------|------|----------|
| UnionID关联失败 | 无法归因用户 | 确保开放平台绑定正确 |
| 订阅消息授权率低 | 触达受限 | 在关键节点引导授权（如支付后） |
| 企微群发频率限制 | 营销受限 | 分批发送，结合私聊补充 |

### 7.2 合规风险

| 风险点 | 影响 | 应对方案 |
|--------|------|----------|
| 订阅消息内容违规 | 模板被拒 | 避免营销用语，强调服务通知 |
| 小程序审核不通过 | 上线延迟 | 参考2026审核新规，注重品质和体验 |

### 7.3 建议

1. **优先级**：先跑通核心流程（拉新→交易→履约），再完善营销功能
2. **成本控制**：初期选择最低成本配置，业务量上来后再升级
3. **用户体验**：订阅消息授权放在支付成功后，转化率最高
