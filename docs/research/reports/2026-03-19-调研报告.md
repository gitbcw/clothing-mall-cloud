## 一、微信授权生日字段调研

### 1.1 调研结论

**微信小程序 API 不返回用户生日信息**

### 1.2 getUserProfile 返回字段

根据 [微信官方文档 - UserInfo](https://developers.weixin.qq.com/miniprogram/dev/api/open-api/user-info/UserInfo.html)，`userInfo` 对象包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| `nickName` | string | 用户昵称 |
| `avatarUrl` | string | 用户头像 URL |
| `gender` | number | 性别（0:未知, 1:男, 2:女） |
| `country` | string | 国家 |
| `province` | string | 省份 |
| `city` | string | 城市 |
| `language` | string | 语言 |

**❌ 不包含 `birthday`（生日）字段**

### 1.3 重要变化（2022年11月起）

自 2022年11月8日 起，`wx.getUserProfile` 接口已被调整：
- 获取用户头像统一返回**默认灰色头像**
- 昵称统一返回**"微信用户"**

**新方式获取用户信息**：
- 头像：使用 `<button open-type="chooseAvatar">` 让用户选择
- 昵称：使用 `<input type="nickname">` 让用户填写

### 1.4 获取生日的方案

**唯一方案：用户主动填写表单**

✅ 已实现：登录后弹出生日填写弹窗

## 二、快递运费调研

### 2.1 主流计费方式

#### 按件计费（首重+续重）

**通用公式**：`首重运费 + (总重量 - 首重) × 续重单价`

**示例（极兔快递）**：
| 地区 | 首重(1kg) | 续重(每kg) |
|------|-----------|------------|
| 华南/华中/华东 | 12元 | 11元 |
| 其他地区 | 15元 | 12元 |

#### 按重量计费

**体积重量计算**：
| 快递公司 | 计算公式 |
|----------|----------|
| 中通 | 长×宽×高 ÷ **6000** |
| 申通 | 长×宽×高 ÷ **8000** |
| DHL | 长×宽×高 ÷ **5000** |

**计费原则**：取体积重量与实际重量的**较大值**

#### 固定运费

全国统一价格，适合小件服装店简化管理。

#### 满额包邮

满 X 元包邮，常见促销策略。

### 2.2 运费模板设计建议

**推荐方案（服装店适用）**：

| 配置项 | 建议值 | 说明 |
|--------|--------|------|
| 计费方式 | **按件计费** | 服装重量差异小，按件简单 |
| 首重价格 | 10-12元 | 参考极兔、中通商家价 |
| 续重价格 | 2-5元/kg | 视合作快递而定 |
| 包邮门槛 | 满99/199元 | 常见电商标准 |
| 偏远地区 | 加收5-10元 | 新疆、西藏、内蒙古 |

### 2.3 系统需实现功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 运费模板配置 | 按地区、件数计算 | P0 |
| 包邮规则 | 满X元包邮 | P0 |
| 前端展示 | 小程序显示运费明细 | P1 |
| 结算集成 | 订单结算时自动计算 | P0 |

### 2.4 快递合作建议

**推荐优先洽谈**：
1. **极兔速递** - 价格较低，适合小商家
2. **中通快递** - 覆盖广，服务稳定
3. **圆通快递** - 价格适中

**合作流程**：
1. 联系当地快递网点
2. 提供日均发货量预估
3. 商谈月结价格
4. 签订合作协议

## 三、生日信息隐私合规调研

### 3.1 法律法规要求

**《个人信息保护法》核心原则**：
- **最小必要**：收集应限于实现处理目的的最小范围
- **告知同意**：收集前需说明目的、方式、范围
- **目的限制**：不得超出约定范围使用

### 3.2 2025年监管重点

根据 [金杜律师事务所 - 2025年APP监管趋势](https://www.kwm.com/cn/zh/insights/latest-thinking/trend-of-supervision-over-mobile-applications-in-china-2025-and-compliance-implications.html)：

> "强制收集非必要的手机号、生日、性别等信息"是 2025 年重点监管问题

### 3.3 微信小程序平台规范

根据 [微信开放社区](https://developers.weixin.qq.com/community/develop/doc/000ee86d9a8a401fd88160f0766809)：

> "除非相关法律要求或经用户同意，不得要求用户输入个人信息（包括出生日期）才可使用功能"

### 3.4 合规建议

| 要求 | 实现方式 | 状态 |
|------|----------|------|
| **非强制收集** | 提供「跳过」按钮 | ✅ 已实现 |
| **告知目的** | 弹窗说明"生日享受七折优惠" | ✅ 已实现 |
| **用户同意** | 点击「确认」才算同意 | ✅ 已实现 |
| **隐私政策** | 更新隐私协议，说明生日用途 | ⚠️ 待补充 |
| **最小化保存** | 仅保存生日日期，不保存其他敏感信息 | ✅ 已实现 |

### 3.5 建议补充事项

1. **更新隐私政策**：
   - 在《隐私协议》中增加生日信息收集条款
   - 说明收集目的：生日权益发放
   - 说明保存期限：账户存续期间

2. **用户协议弹窗**：
   - 首次使用时展示隐私协议
   - 用户同意后方可使用

3. **数据安全**：
   - 生日信息不对外展示
   - 仅用于生日权益发放
