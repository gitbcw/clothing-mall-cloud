## 项目背景

服装店线上渠道扩展系统，基于 Spring Boot 后端 + Vue 管理前端 + 微信小程序用户前端。

**核心目标**：
1. ✅ 小程序管理端扩展 — 手机端快速上架、订单处理
2. ✅ 菜单重构 — 9 菜单 → 5 菜单（后台），7 菜单 → 5 菜单（管理端）
3. ✅ SKU 管理 — 拍照录入 → SKU 库 → 选择上架
4. ✅ 管理端 TabBar 重构 — 9 页面 → 双 Tab 结构
5. ✅ 管理端业务流程打通 — 前后端数据完整闭环

## 小程序管理端结构（当前）

```
管理端 TabBar（底部双 Tab）
├── 📋 订单 Tab (tabOrder)
│   ├── 统计概览卡片（待处理订单 / 售后中 / 待上架商品）
│   ├── 订单子 Tab — 待处理订单（待确认/待发货/退款中/待核销）
│   │   └─ 卡片操作：确认接单 / 发货 / 同意退款 / 拒绝退款 / 核销
│   └── 售后子 Tab — 退款中订单
│
└── 📦 上架 Tab (tabShelf)
    ├── 商品上架子 Tab — 内嵌表单（拍照/AI/手动填写）
    │   └─ 字段：名称、价格(专柜/零售/特价)、分类、场景标签、简介、详情、关键词、参数
    │   └─ 操作：预览 / 保存草稿 / 立即上架
    └── 商品列表子 Tab — 在售/待上架筛选 + 搜索
        └─ 操作：管理模式(批量选择) / 编辑 / 上架 / 下架 / 批量删除 / 一键全部下架

独立页面（从卡片跳转）：
├── 订单详情 (orderDetail) — 发货弹窗 / 核销弹窗
└── 商品编辑 (goodsEdit) — 完整编辑（与上架表单字段对齐）
```
