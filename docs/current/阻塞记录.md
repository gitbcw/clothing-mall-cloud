## 阻塞记录：2.2.1 - 多模态 LLM API 选型

**阻塞时间**: 2026-03-21
**阻塞原因**: 需要付费的 AI API Key 才能使用真实的服装识别功能
**所需资源**:
- 方案一：MiniMax API Key（推荐，国内可用）
- 方案二：OpenAI API Key（GPT-4 Vision）
- 方案三：百度文心一言 API Key
- 方案四：阿里通义千问 API Key

**已完成的替代方案**:
- 创建了 `WxAiController.java` 提供 Mock 识别接口
- Mock 接口返回随机生成的识别结果，可用于前端开发测试
- 创建了 `AiConfig.java` 配置类，支持通过 application.yml 配置 AI 服务

**临时解决方案**:
- 前端调用 `/wx/ai/recognizeByUrl` 时，返回 Mock 数据
- 响应中包含 `isMock: true` 标识，前端可据此提示用户

**恢复方案**:
1. 获取 AI API Key（如 MiniMax）
2. 在 `application-core.yml` 中配置：
   ```yaml
   litemall:
     ai:
       enabled: true
       provider: minimax
       api-key: 你的API Key
       endpoint: https://api.minimax.io/v1/chat/completions
       model: MiniMax-Text-01
   ```
3. 在 `WxAiController` 中实现真实的 AI 调用逻辑
